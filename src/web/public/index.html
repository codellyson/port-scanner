Port Scanner

A lean CLI and web dashboard to list active ports and kill processes on your system.

GitHub

Why This Package

If you juggle multiple projects at once, you know the pain. Dev servers pile up, ports collide, and something you forgot to stop three hours ago is still hogging port 3000.

I built this because I got tired of the ritual: open a terminal, run lsof or netstat, squint at the output, find the PID, then kill it. Every single time.

Port Scanner gives you one command to see every active port and one click to kill what you don't need. That's it. No extras, no ceremony.

Installation

Global (Recommended)

Install globally to use the ports command anywhere.

# npm
npm install -g port-scanner-cli

# pnpm
pnpm add -g port-scanner-cli

# yarn
yarn global add port-scanner-cli

Local

Add to your project as a dependency.

# npm
npm install port-scanner-cli

# pnpm
pnpm add port-scanner-cli

# yarn
yarn add port-scanner-cli

Usage

List all ports

Scan and display all open ports on the system.

ports list

Filter by protocol

Show only TCP or UDP connections.

ports list --protocol tcp
ports list -P udp

Filter by port

Look up a specific port number.

ports list --port 3000

Filter by state

Show only listening or established connections.

ports list --state LISTEN

Filter by process

Find ports used by a specific process.

ports list --process node

JSON output

Get results as JSON for scripting.

ports list --json

Web dashboard

Launch the browser-based dashboard.

ports web
ports web --port 8080

Combine filters

Chain multiple filters together.

ports list --protocol tcp --state LISTEN

Requirements

Node.js >= 18 Linux: ss or netstat macOS: lsof or netstat Windows: netstat
